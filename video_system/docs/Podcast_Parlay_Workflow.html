Recent Change: Captions Before Video Production

Captions approval (video_captions) is now the explicit gate before final video production approval (video_production). This prevents re-work on the polished final video.

Detailed Phases

PHASE 0

Setup (Human + Nathan)

  • • Complete interview in Riverside → export raw tracks + transcript
  • • Run new_project.py to scaffold episode folder
  • • Drop assets (host/guest/b-roll + branding + music)
  • • Tell Nathan: "Start Podcast Parlay for [episode]"
PHASE 1

Planning & First Draft Assembly

Nathan + Alex propose intro, scene cuts, layouts, lower thirds, b-roll placement, and music. Update video_plan.json → run Resolve timeline builder or manual assembly in DaVinci Resolve.
PHASE 2 — KEY GATE

Video Assembly Draft + Captions Approval

Captions must be approved before final video production.

1. Assembly Draft
Render visuals-focused draft (no or temp captions).
2. Captions Generation
Use Resolve or make_template_subtitles.py + AI/Alex polish.
3. Captions Approval (via Teams)
→ request_video_approval(stage="video_captions") — client approves text, timing, and style first.
PHASE 3

Final Video Production + Approval

Once captions are approved, incorporate them into the final Resolve timeline + polish. Then request video_production approval.

action_type = video_production
PHASE 4

Long-Form YouTube Publish

Thumbnail, description, end card → unlisted upload after production approval.

PHASE 5

Clips (5-10 Shorts)

Select moments → generate captioned clips → approval card for the package.

PHASE 6

Clips Publish + Close

Upload each clip + add to playlist. Update project memory.
